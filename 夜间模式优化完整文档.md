# 高校社团管理系统 - 夜间模式优化完整文档

## 一、项目概述

本项目是一个基于 Vue3 + Element Plus 的高校社团管理系统。为了提升用户体验，我们实现了完整的夜间模式（暗黑模式）功能。

### 技术栈
- **前端框架**: Vue 3 (Composition API)
- **UI 组件库**: Element Plus
- **状态管理**: Pinia
- **样式预处理**: SCSS/Sass
- **构建工具**: Vite

---

## 二、夜间模式架构设计

### 2.1 实现方案

我们采用 **Element Plus 官方暗黑模式 + 自定义样式覆盖** 的混合方案：

1. **基础层**: 引入 Element Plus 官方暗黑主题 CSS 变量
2. **覆盖层**: 通过 `dark.scss` 覆盖和扩展 CSS 变量
3. **组件层**: 各页面组件使用 CSS 变量替代硬编码颜色

### 2.2 核心原理

```
┌─────────────────────────────────────────────────────────────┐
│                    夜间模式实现架构                          │
├─────────────────────────────────────────────────────────────┤
│  1. Element Plus 官方暗黑主题 (css-vars.css)                │
│     └── 提供基础暗黑变量: --el-bg-color, --el-text-color    │
│                                                             │
│  2. 自定义覆盖层 (dark.scss)                                 │
│     └── 覆盖布局样式、添加过渡动画                           │
│                                                             │
│  3. 组件样式层                                               │
│     └── 各页面使用 var(--el-xxx) 替代 #fff, #333 等硬编码   │
│                                                             │
│  4. 主题状态管理 (theme.js)                                  │
│     └── 使用 Pinia 管理主题状态，持久化到 localStorage       │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1.3 theme.js - 主题状态管理
**文件路径**: `frontend/src/stores/theme.js`

**核心逻辑**:
```javascript
import { defineStore } from 'pinia'
import { ref, watch } from 'vue'

export const useThemeStore = defineStore('theme', () => {
  // 主题状态: 'light' | 'dark'
  const theme = ref(localStorage.getItem('theme') || 'light')

  // 切换主题
  const toggleTheme = () => {
    theme.value = theme.value === 'light' ? 'dark' : 'light'
  }

  // 监听主题变化，应用到 HTML 元素
  watch(theme, (newTheme) => {
    const html = document.documentElement
    if (newTheme === 'dark') {
      html.classList.add('dark')
    } else {
      html.classList.remove('dark')
    }
    localStorage.setItem('theme', newTheme)
  }, { immediate: true })

  return { theme, toggleTheme }
})
```

**实现要点**:
1. 使用 Pinia 管理主题状态
2. 通过 `localStorage` 持久化用户选择
3. 通过给 `<html>` 元素添加/移除 `.dark` 类来切换主题
4. `immediate: true` 确保页面加载时立即应用保存的主题

## 五、开发规范

### 5.1 样式编写规范

#### ✅ 正确做法
```scss
// 使用 CSS 变量
.my-component {
  background: var(--el-bg-color);
  color: var(--el-text-color-primary);
  border: 1px solid var(--el-border-color);
  
  &:hover {
    background: var(--el-fill-color-light);
  }
```

```scss
// 硬编码颜色
.my-component {
  background: #fff;
  color: #303133;
  border: 1px solid #dcdfe6;
  
  &:hover {
    background: #f5f7fa;
  }
}
```

## 六、常见问题排查

### 6.1 页面仍有白色区域

**排查步骤**:
1. 打开浏览器开发者工具 (F12)
2. 使用元素选择器点击白色区域
3. 查看该元素的 `background` 或 `background-color` 属性
4. 如果是硬编码颜色（如 `#fff`, `#ffffff`, `#f8fafc`），需要修改为 CSS 变量

### 6.2 文字看不清

**原因**: 文字颜色与背景对比度不足

**解决方案**:
- 主要文字使用 `var(--el-text-color-primary)`
- 确保深色模式下文字颜色为浅色（#e0e0e0 左右）
- 避免使用 `opacity` 降低文字可见度

### 6.3 主题切换不流畅

**解决方案**:
确保 `dark.scss` 中的过渡动画已生效：
```scss
* {
  transition: background-color 0.3s ease, 
              border-color 0.3s ease, 
              color 0.3s ease,
              box-shadow 0.3s ease;
}
```

## 七、后续优化建议

### 7.1 自动化检测
可以考虑添加 ESLint 或 Stylelint 规则，禁止在样式中使用硬编码颜色值。

### 7.2 主题扩展
如需支持更多主题（如高对比度模式、色盲友好模式），可以扩展 CSS 变量系统。

### 7.3 系统主题同步
可以监听系统主题变化，自动切换：
```javascript
window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', (e) => {
  themeStore.theme = e.matches ? 'dark' : 'light';
});
```

# 社团管理系统 - 系统架构说明

> 本文档用于论文撰写参考及答辩准备

## 一、总体架构

系统采用**前后端分离架构**，后端基于 SpringBoot 构建 RESTful API，前端使用 Vue3 渲染页面，通过 HTTP 请求进行数据交互。

```
┌─────────────────────────────────────────────────────────┐
│                     前端 (Vue3)                          │
│            views → api → stores → components             │
└───────────────────────────┬─────────────────────────────┘
                            │ HTTP Request / JSON
                            ▼
┌─────────────────────────────────────────────────────────┐
│              后端 (SpringBoot)                           │
│                                                         │
│   ┌─────────────────────────────────────────────────┐   │
│   │           接口层 (Controller)                    │   │
│   │    接收请求 → 参数校验 → 调用业务层 → 返回响应     │   │
│   └──────────────────────┬──────────────────────────┘   │
│                          │                              │
│                          ▼                              │
│   ┌─────────────────────────────────────────────────┐   │
│   │            业务层 (Service)                      │   │
│   │      核心业务逻辑 → 事务控制 → 数据组装          │   │
│   └──────────────────────┬──────────────────────────┘   │
│                          │                              │
│             ┌────────────┴────────────┐                 │
│             ▼                         ▼                 │
│   ┌─────────────────┐   ┌─────────────────────────┐     │
│   │  数据访问层      │   │      数据模型层          │     │
│   │   (Mapper)      │   │   (Entity / DTO / VO)   │     │
│   │                 │   │                         │     │
│   │  数据库 CRUD    │   │  各层数据传递与封装      │     │
│   └────────┬────────┘   └─────────────────────────┘     │
│            │                                            │
└────────────┼────────────────────────────────────────────┘
             ▼
┌─────────────────────────────────────────────────────────┐
│                   数据库 (MySQL)                         │
└─────────────────────────────────────────────────────────┘

        ┌──────────────────────────────────────────┐
        │           支撑层 (横切关注点)              │
        │  security │ config │ annotation          │
        │  aspect   │ common │ enums │ exception   │
        │                                          │
        │   横穿所有层，为各层提供基础支撑能力        │
        └──────────────────────────────────────────┘
```

## 二、分层架构详解

### 1. 接口层 (Controller)

- **职责**：接收前端 HTTP 请求，进行参数校验，调用业务层处理，返回统一格式的 JSON 响应
- **包含**：10 个 Controller
  - AuthController — 登录、注册
  - UserController — 用户管理
  - ClubController — 社团管理
  - ActivityController — 活动管理
  - ApplyController — 入团申请
  - PaymentController — 费用管理
  - NotificationController — 通知公告
  - DashboardController — 数据统计
  - PresidentTransferController — 社长交接
  - SystemLogController — 系统日志

### 2. 业务层 (Service)

- **职责**：处理核心业务逻辑，控制事务，组装数据
- **包含**：9 个 Service 接口及对应实现类
- **特点**：所有业务规则在此层实现，确保 Controller 轻量、Mapper 纯粹

### 3. 数据层

数据层分为**数据访问层**和**数据模型层**两个子层：

#### 3.1 数据访问层 (Mapper)

- **职责**：负责与数据库直接交互，执行增删改查操作
- **技术**：MyBatis-Plus，通过 Mapper 接口和 XML 映射实现
- **包含**：13 个 Mapper

#### 3.2 数据模型层 (Entity / DTO / VO)

- **职责**：数据在各层之间的传递与封装，实现层间解耦
- **包含**：

| 类型 | 说明 | 数量 |
|------|------|------|
| Entity | 与数据库表一一对应的实体类 | 14 个 |
| DTO (Data Transfer Object) | 接收前端请求参数的入参封装 | 8 个 |
| VO (View Object) | 返回给前端的视图对象，用于脱敏和数据聚合 | 7 个 |

> **设计优势**：Entity 不直接暴露给前端，避免敏感字段（如密码）泄露，同时 VO 可以聚合多个实体的数据，减少前端请求次数。

### 4. 支撑层

- **职责**：横切关注点，为所有层提供基础能力，不参与核心业务流程
- **包含**：

| 包 | 说明 |
|----|------|
| security/ | JWT 认证、登录过滤器、用户详情服务 |
| config/ | MyBatis-Plus 配置、安全配置、自动填充 |
| annotation/ | 自定义注解（如操作日志注解） |
| aspect/ | AOP 切面（日志记录等） |
| common/ | 统一返回结果 Result、分页结果 PageResult |
| enums/ | 业务枚举（角色、状态等） |
| exception/ | 全局异常处理器 |

## 三、核心请求流转路径

以「用户创建社团」为例：

```
用户填写社团信息
       │
       ▼
前端 axios.post('/api/club', formData)
       │
       ▼
ClubController.create(@RequestBody ClubDTO dto)
       │  ① 接收请求，校验参数
       │  ② 获取当前登录用户信息
       ▼
ClubService.create(ClubDTO dto)
       │  ③ 业务校验（社团名是否重复等）
       │  ④ DTO → Entity 转换
       │  ⑤ 设置默认值（社长、状态等）
       ▼
ClubMapper.insert(Club club)
       │  ⑥ 执行 INSERT SQL
       ▼
MySQL 写入数据
       │
       ▼
返回 Result<ClubVO> 给前端
```

## 四、分层架构优势

1. **职责分离**：各层专注于自身职责，接口层只管请求响应，业务层只管逻辑，数据层只管数据库，便于团队协作开发

2. **低耦合设计**：某一层内部修改不影响其他层，例如更换数据库只需修改 Mapper 层，调整前端交互只需修改 Controller 层

3. **可维护性强**：代码结构清晰，问题定位快速，便于后续功能扩展

4. **代码复用率高**：通用逻辑集中在支撑层实现（如 JWT 认证、统一响应、日志记录），避免重复代码

5. **数据安全**：通过 DTO/VO 分离，Entity 不直接暴露给前端，有效防止敏感数据泄露

## 五、技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue3 + Element Plus + Pinia + Axios + Vue Router |
| 后端 | SpringBoot 2.7 + Spring Security + JWT |
| 数据库 | MySQL 8.0 |
| ORM | MyBatis-Plus |
| 构建 | Maven (后端) + Vite (前端) |

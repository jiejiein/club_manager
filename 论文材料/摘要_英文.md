# Abstract

Student clubs serve as important carriers of campus culture construction with their quantity and scale growing year by year while traditional manual management methods cannot meet the requirements of information centralization and process standardization. This project addresses the problems of dispersed information, inefficient approval processes and lagging data statistics in current university club management and designs and implements a university club management and activity reservation platform based on B/S architecture. The system adopts SpringBoot framework to build backend services, uses MyBatis-Plus to implement data persistence and MySQL to store business data. User authentication and authorization are completed through Spring Security combined with JWT token, and role-based access control model is adopted to implement three-level permission control for administrators, club presidents and ordinary users. The frontend is developed based on Vue3 Composition API, uses Element Plus component library to build user interface, adopts Pinia for state management and completes data visualization through ECharts. The system includes seven functional modules: user management, club management, activity management, membership application approval, notification announcement, fee payment and data dashboard, covering the full lifecycle management of clubs from establishment to operation. The database design contains 13 core data tables covering business data such as user information, club information, activity information, approval records and payment records. The system implements online and standardized management of core business processes including club creation and review, activity publishing and registration, membership application and approval. It supports dark mode switching to improve user visual experience and has responsive layout capability to adapt to multi-terminal access. Functional testing and performance testing verify that the system response time is controlled within 2 seconds, supports concurrent access of hundreds of users, and all core functions operate normally meeting the design requirements. The construction of this platform provides a standardized information solution for university club management, reduces manual management costs and improves business processing efficiency with certain practical application value.

**Keywords:** University Club Management; SpringBoot; Vue3; B/S Architecture; RBAC

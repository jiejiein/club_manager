## 三、数据库设计类

### 4. E-R 图（实体关系图）（必备 ✅）

核心实体及其关系：

```
┌──────────┐         ┌──────────┐         ┌──────────┐
│   用户    │  N : N  │   社团    │  1 : N  │   活动    │
│  (User)  │◄───────►│  (Club)  │────────►│(Activity)│
└──────────┘  加入   └──────────┘  发起   └──────────┘
     │                     │                    │
     │ 1:N                 │ 1:N                │ N:N
     ▼                     ▼                    ▼
┌──────────┐         ┌──────────┐         ┌──────────┐
│ 入团申请  │         │   费用    │         │  活动报名  │
│ (Apply)  │         │(Payment) │         │(SignUp)  │
└──────────┘         └──────────┘         └──────────┘
```

完整实体关系：

| 实体 | 关联实体 | 关系 |
|------|---------|------|
| 用户 | 社团 | N:N（通过 ClubMember） |
| 用户 | 入团申请 | 1:N |
| 用户 | 活动报名 | 1:N |
| 社团 | 社团类型 | N:1 |
| 社团 | 活动 | 1:N |
| 社团 | 社长(用户) | N:1 |
| 活动 | 活动报名 | 1:N |
| 社团 | 费用 | 1:N |
| 社团 | 通知 | 1:N |

### 5. 数据库表结构（必备 ✅）

每张核心表画一个表格（论文中不需要画图，用表格即可）：

**sys_user（用户表）**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 用户ID，主键 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(255) | 密码 |
| nickname | VARCHAR(50) | 昵称 |
| role | INT | 角色：1管理员 2社长 3普通用户 |
| status | INT | 状态：0禁用 1启用 |
| student_no | VARCHAR(50) | 学号 |
| college | VARCHAR(100) | 学院 |
| create_time | DATETIME | 创建时间 |
| deleted | INT | 逻辑删除 |

（其余表同理：sys_club、sys_activity、sys_apply、sys_payment、sys_notification、sys_club_member、sys_club_type、sys_activity_signup）

## 四、系统设计类

### 6. 核心业务流程图（必备 ✅）

**社团创建流程：**
```
    ┌─────────┐
    │  开始    │
    └────┬────┘
         ▼
    ┌─────────────┐
    │ 填写社团信息  │
    └──────┬──────┘
           ▼
    ┌─────────────┐     ┌──────────┐
    │ 信息校验     │────→│ 提示错误  │──→ 返回修改
    └──────┬──────┘     └──────────┘
           │ 通过
           ▼
    ┌─────────────┐
    │ 提交申请     │
    └──────┬──────┘
           ▼
    ┌─────────────┐
    │ 管理员审核   │
    └──────┬──────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌────────┐  ┌────────┐
│ 审核通过 │  │ 审核拒绝 │
└────┬───┘  └────┬───┘
     ▼           ▼
┌────────┐  ┌────────┐
│ 社团成立 │  │ 通知用户 │
└────┬───┘  └────┬───┘
     ▼           ▼
    ┌─────────┐
    │  结束    │
    └─────────┘
```

**活动报名流程：**
```
社长发布活动 → 管理员审核 → 活动上线 → 用户浏览 → 用户报名 → 报名成功 → 参加活动
```

**入团申请流程：**
```
用户申请入团 → 社长审核 → 审核通过 → 加入社团成员列表
```
